## Zustand Store 职责边界与协作

### Store 整体架构

根据 [tech-arch.md](file:///Users/tog/Desktop/code/solo/xyj-116/.trae/documents/tech-arch.md) 的设计，项目采用「领域驱动」的状态管理模式，将状态按业务领域拆分为多个独立的 store。核心三个为：

| Store | 核心职责 | 所属领域 |
|--------|----------|----------|
| `useAnatomyStore` | 解剖层次与身体系统 | 业务领域 |
| `useSelectionStore` | 结构选中与悬停 | 交互领域 |
| `useViewStore` | 相机与视图设置 | 视图领域 |

此外，项目还扩展了 6 个辅助 store，整体形成「3 核心 + 6 辅助」的分布式状态管理体系。

### Store 设计原则总结

1. **单一职责**：每个 Store 只管理一个明确的业务领域
2. **独立可组合**：组件按需订阅多个 Store，避免 props 层层传递
3. **单向数据流**：UI → Action → State → UI
4. **跨 Store 通信**：通过 `get()` 方法访问其他 Store 状态
5. **开发友好**：关键 Store 挂载到 `window` 便于调试

## React Three Fiber (R3F) 工作原理

### R3F 本质：React 渲染器 for Three.js

R3F 不是对 Three.js 的「封装」，而是一个**自定义 React 渲染器**，它将 React 的声明式组件模型映射到 Three.js 的命令式 API。

```
React 组件树 (JSX)
        │
        ▼
┌─────────────────────────┐
│  R3F 渲染器     │
│  (Custom Renderer)  │
└─────────┬───────────────┘
          │
          ▼
Three.js 场景图 (Scene Graph)
        │
        ▼
WebGL 绘制调用
```

### R3F 核心转换原理

#### 1. JSX → Three.js 对象映射

R3F 使用 React 的 `reconciler`（协调器）机制，将 JSX 元素转换为 Three.js 对象实例。

**转换示例（本项目）：

| JSX 元素 | 对应 Three.js 对象 | 代码位置 |
|-----------|-------------------|----------|
| `<Canvas>` | `THREE.WebGLRenderer + THREE.Scene + THREE.Camera | [AnatomyScene.tsx#L212-L234](file:///Users/tog/Desktop/code/solo/xyj-116/src/components/three/AnatomyScene.tsx#L212-L234) |
| `<mesh>` | `THREE.Mesh` | [Organ.tsx#L238-L247](file:///Users/tog/Desktop/code/solo/xyj-116/src/components/three/Organ.tsx#L238-L247) |
| `<group>` | `THREE.Group` | [Organ.tsx#L232-L256](file:///Users/tog/Desktop/code/solo/xyj-116/src/components/three/Organ.tsx#L232-L256) |
| `<boxGeometry>` | `THREE.BoxGeometry` | [Organ.tsx#L16-L31](file:///Users/tog/Desktop/code/solo/xyj-116/src/components/three/Organ.tsx#L16-L31) |
| `<meshStandardMaterial>` | `THREE.MeshStandardMaterial` | [Organ.tsx#L78-L124](file:///Users/tog/Desktop/code/solo/xyj-116/src/components/three/Organ.tsx#L78-L124) |
| `<color>` | 场景背景色 | [AnatomyScene.tsx#L220](file:///Users/tog/Desktop/code/solo/xyj-116/src/components/three/AnatomyScene.tsx#L220) |

**转换原理**：
- R3F 识别小写 JSX 标签（如 `<mesh>`），自动实例化对应的 Three.js 类（如 `new THREE.Mesh()`）
- JSX 属性自动映射到对象属性
- 子元素自动作为父元素的 `children` 或通过 `attach` 到特定属性

#### 2. 自动清理与生命周期

```
组件挂载 → 实例化 Three.js 对象 → 添加到父场景 → 属性更新 → 组件卸载 → 从场景移除 → dispose() 释放资源
```

**本项目中的体现**：
- [Organ.tsx#L67](file:///Users/tog/Desktop/code/solo/xyj-116/src/components/three/Organ.tsx#L67) `useMemo` 创建几何体，随 `structure.geometry.type` 变化时自动重建
- [Organ.tsx#L78-L124](file:///Users/tog/Desktop/code/solo/xyj-116/src/components/three/Organ.tsx#L78-L124) `useMemo` 创建材质，依赖变化时自动重新创建
- R3F 自动处理 `dispose` 释放 WebGL 资源

#### 3. useFrame：帧循环 Hook

`useFrame` 是 R3F 的核心 Hook，将 React 注册渲染循环。

**原理**：
```typescript
useFrame((state, delta) => {
  // 每帧调用 ~60fps
})
```

- 自动绑定到 Three.js 的 `requestAnimationFrame` 循环
- 提供 `state` 包含 `clock`, `camera`, `scene`, `raycaster` 等
- 自动在组件卸载时取消注册

**本项目应用**：

[Organ.tsx#L147-L181](file:///Users/tog/Desktop/code/solo/xyj-116/src/components/three/Organ.tsx#L147-L181)

```typescript
useFrame((state) => {
  if (meshRef.current) {
    if (flashEffect === 'correct') {
      // 正确答案弹跳动画
      const bounceScale = 1 + Math.sin(state.clock.getElapsedTime() * 8) * 0.1;
      meshRef.current.scale.setScalar(bounceScale);
    } else if (flashEffect === 'wrong') {
      // 错误答案抖动动画
      const shakeOffset = Math.sin(state.clock.getElapsedTime() * 20) * 0.05;
      meshRef.current.position.x = shakeOffset;
    } else if (isSearchHighlighted) {
      // 搜索高亮脉动
      const pulseScale = 1 + Math.sin(state.clock.getElapsedTime() * 5) * 0.06;
      meshRef.current.scale.setScalar(pulseScale);
    }
    // ... 更多动画逻辑
  }
  
  // 心脏独立的心跳动画
  if (groupRef.current && structure.id === 'heart') {
    const pulse = 1 + Math.sin(state.clock.getElapsedTime() * 2) * 0.05;
    groupRef.current.scale.setScalar(pulse);
  }
```

**设计亮点**：
- 单一 `useFrame` 处理所有动画逻辑
- 使用 `state.clock.getElapsedTime() 获取统一时间源
- 条件分支实现不同状态下的不同动画

#### 6. 声明式 vs 命令式对比

| 方面 | 纯 Three.js (命令式) | R3F (声明式) |
|------|---------------------|-------------|
| 对象创建 | `const mesh = new THREE.Mesh(geo, mat)` | `<mesh geometry={geo} material={mat} />` |
| 属性更新 | `mesh.position.set(x, y, z)` | `<mesh position={[x, y, z]} />` |
| 动画循环 | `renderer.setAnimationLoop(callback)` | `useFrame(callback)` |
| 事件处理 | 手动 raycaster 实现 | `<mesh onClick={handler} />` |
| 资源清理 | 手动 `geometry.dispose()` | 自动 |

## 总结

### Store 设计模式

本项目采用「领域驱动的 zustand 状态管理架构：

- **3 个核心 Store 关注「是什么（解剖层次）、「选什么（选中交互）、「怎么看（视图控制」三大领域
- **6 个辅助 Store 扩展搜索、测验、标注、孤立查看等功能
- Store 之间通过 `get()` 实现跨 Store 访问，组件通过订阅多个 Store 实现状态联动
- 单向数据流保证状态变更 → UI 响应的清晰数据流

### R3F 转换原理

R3F 本质上是一个 React 自定义渲染器，核心机制：

1. **JSX → 对象映射**：将声明式组件 → Three.js 对象
2. **useFrame 帧循环**：注册每帧执行回调，实现动画
3. **useThree 上下文**：访问 Three.js 核心对象
4. **事件系统**：自动 Raycaster 拾取，将 DOM 事件映射到 3D 对象
5. **自动生命周期**：挂载/卸载自动管理 Three.js 资源

本项目中 R3F 与命令式 Three.js 结合使用：
- **JSX 用于场景结构组织
- 复杂材质/几何体形使用命令式 Three.js API
- useFrame 实现多状态动画逻辑
- useMemo 缓存避免重复创建

这种「声明式 + 命令式」的混合模式，既发挥了 React 组件化优势，又保留了 Three.js 的灵活控制能力。
